
# 03 - Security Assessment

> Version: 0.5

# Learning Objectives

After this module you should be able to:

- Explain the purpose of a security assessment.
- Distinguish assessments from penetration tests and audits.
- Understand common assessment methods.
- Select the appropriate assessment for a given CISSP scenario.

---

# 1. Assessment Lifecycle

```text
Define Scope
     │
     ▼
Identify Assets
     │
     ▼
Select Assessment Method
     │
     ▼
Collect Evidence
     │
     ▼
Analyze Findings
     │
     ▼
Report Results
     │
     ▼
Remediate
     │
     ▼
Reassess
```

The goal is **continuous improvement**, not simply producing a report.

---

# 2. Types of Security Assessments

| Assessment | Goal | Typical Output |
|------------|------|----------------|
| Vulnerability Assessment | Identify weaknesses | Vulnerability report |
| Configuration Assessment | Verify secure configuration | Configuration findings |
| Compliance Assessment | Verify regulatory or policy compliance | Compliance report |
| Risk Assessment | Estimate business risk | Risk register |
| Penetration Test | Validate exploitability | Exploitation report |

---

# 3. Assessment vs Audit vs Penetration Test

| Assessment | Audit | Penetration Test |
|------------|-------|------------------|
| Measures security posture | Measures compliance | Demonstrates exploitability |
| Broad | Formal | Focused |
| May include scanning | Reviews evidence | Attempts controlled attack |

## Manager Mindset

Ask yourself:

> "What question is management asking?"

- "Are we compliant?" → Audit
- "Where are our weaknesses?" → Assessment
- "Can this weakness actually be exploited?" → Penetration Test

---

# 4. Evidence Quality

Good evidence is:

- Objective
- Repeatable
- Verifiable
- Relevant
- Sufficient

Poor evidence creates poor decisions.

---

# 5. Reporting Principles

A good report answers:

1. What was assessed?
2. What was found?
3. Why does it matter?
4. What is the business impact?
5. What should be done next?

---

# 6. CISSP Exam Traps

| Question mentions... | Think... |
|----------------------|----------|
| Compliance | Audit |
| Weakness identification | Assessment |
| Proof of exploitation | Penetration Test |
| Executive decision | Risk and business impact |
| Technical details only | Probably not the best CISSP answer |

# 30-Second Revision

- Assessment measures security.
- Audit measures compliance.
- Penetration testing proves exploitability.
- Evidence supports assurance.
- Reports should drive remediation and management decisions.
