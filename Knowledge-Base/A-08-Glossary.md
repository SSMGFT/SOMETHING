
# Appendix A-08 - Glossary

> Version: 2.1

A concise reference for important CISSP Domain 6 terms.

| Term | Definition |
|------|------------|
| Assurance | Confidence, supported by objective evidence, that security controls are effective. |
| Assessment | Process of evaluating security controls and identifying weaknesses. |
| Audit | Formal evaluation against defined requirements, standards, or regulations. |
| Review | Examination of documents, processes, or activities. |
| Inspection | Detailed examination of configurations or implementations. |
| Evidence | Objective information used to support conclusions. |
| Verification | Confirms the system was built correctly according to specifications. |
| Validation | Confirms the correct solution was built for the business need. |
| Discovery | Identifies live hosts or reachable systems. |
| Enumeration | Extracts detailed information from discovered systems. |
| Banner Grabbing | Identifies software versions exposed by a service. |
| Fingerprinting | Identifies operating systems, devices, or applications. |
| Vulnerability Assessment | Identifies security weaknesses without exploiting them. |
| Credentialed Scan | Authenticated scan with internal visibility. |
| False Positive | Incorrectly reports a vulnerability. |
| False Negative | Fails to detect a real vulnerability. |
| Penetration Test | Controlled attempt to validate exploitability. |
| Rules of Engagement (RoE) | Agreed scope, authorization, and constraints for a penetration test. |
| Static Testing | Evaluates software without execution. |
| Dynamic Testing | Evaluates software while it is running. |
| Fuzz Testing | Uses unexpected or random inputs to identify failures. |
| Mutation Testing | Measures test-suite effectiveness using artificial code changes. |
| Regression Testing | Confirms changes did not break existing functionality. |
| Log | Record of system or application activity. |
| Event | Noteworthy occurrence recorded in logs. |
| Alert | Notification requiring investigation. |
| Incident | Confirmed adverse security event. |
| SIEM | Security Information and Event Management; centralizes and correlates security events. |
| Baseline | Established definition of normal behavior. |
| Continuous Monitoring | Ongoing observation of security controls and events. |
| SOC Report | Independent report describing controls at a service organization. |
| KPI | Key Performance Indicator; measures performance. |
| KRI | Key Risk Indicator; measures risk exposure. |
| Executive Report | Business-focused summary supporting management decisions. |

# Final Reminder

For Domain 6, remember the progression:

```text
Controls
   ↓
Assessment
   ↓
Evidence
   ↓
Assurance
   ↓
Decision
   ↓
Improvement
```

Most CISSP Domain 6 questions can be solved by identifying **where the scenario fits in this lifecycle**.
