
# Appendix A-03 - Memory Hooks

> Version: 1.6

# Purpose

Use these memory hooks during the last few days before the CISSP exam.

---

# Assessment Family

| Concept | Memory Hook |
|---------|-------------|
| Assessment | **Find** weaknesses |
| Audit | **Verify** compliance |
| Penetration Test | **Prove** exploitability |
| Assurance | **Confidence from evidence** |
| Evidence | **Facts before decisions** |

---

# Software Testing

| Concept | Memory Hook |
|---------|-------------|
| Static | Code **Stops** (not running) |
| Dynamic | Code **Drives** (running) |
| Unit | **One** component |
| Integration | Components **Interact** |
| System | **Entire** application |
| Acceptance | **Business approves** |
| Regression | **Nothing broke** |
| Fuzzing | **Random inputs** |
| Mutation | **Break the tests, not the app** |

---

# Monitoring

| Concept | Memory Hook |
|---------|-------------|
| Log | Record |
| Event | Something happened |
| Alert | Investigate |
| Incident | Confirmed problem |
| SIEM | **Correlates** logs |
| Baseline | Normal |
| Continuous Monitoring | Always watching |

---

# Manager Thinking

Remember this order:

```text
Business
   ↓
Risk
   ↓
Policy
   ↓
Assessment
   ↓
Evidence
   ↓
Decision
   ↓
Technology
```

Never choose technology before understanding business risk.

---

# Five Golden Rules

1. Assessment **finds**.
2. Pen Test **proves**.
3. Audit **verifies**.
4. SIEM **correlates**.
5. Executives want **business impact**, not technical detail.
