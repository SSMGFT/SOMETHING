# 00 - Learning Path

> Version: 0.2

# Purpose

This roadmap explains **why Domain 6 topics are learned in this order**.
The sequence is based on conceptual dependencies rather than the order of the CISSP Official Study Guide.

---

# Core Question

Every concept in Domain 6 ultimately answers one management question:

> **Can management trust that security controls are working as intended?**

Everything else in this domain supports answering that question.

# Module Roadmap

| Order | Module | Why it comes here |
|------:|--------|-------------------|
| 1 | Foundations | Build the vocabulary used everywhere else. |
| 2 | Security Assessment | Learn what is being measured and why. |
| 3 | Scanning & Enumeration | Learn how systems are discovered and identified. |
| 4 | Vulnerability Assessment | Identify weaknesses without exploiting them. |
| 5 | Penetration Testing | Validate whether weaknesses are exploitable. |
| 6 | Application Security Testing | Verify software quality and security. |
| 7 | Logging & Monitoring | Observe and detect security events. |
| 8 | Audit & Compliance | Produce objective evidence for stakeholders. |
| 9 | Metrics & Reporting | Turn technical findings into management information. |
|10 | Manager Mindset | Apply ISC² decision-making principles. |

# Study Strategy

For every module:

1. Understand the mental model.
2. Learn the terminology.
3. Compare similar concepts.
4. Study real-world examples.
5. Solve mapped Chapter 6 questions.
6. Review the summary before moving on.

Do **not** memorize tools before understanding the process they support.

---

# Completion Criteria

You have mastered a module when you can:

- Explain the purpose of every concept.
- Draw its relationship to neighboring concepts.
- Distinguish it from commonly confused topics.
- Answer scenario-based CISSP questions without relying on memorized keywords.
