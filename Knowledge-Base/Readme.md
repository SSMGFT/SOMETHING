# CISSP Domain 6 Knowledge Base

> **Version:** 0.1
> **Domain:** Domain 6 – Security Assessment and Testing
> **Objective:** Build conceptual understanding rather than memorization.

---

# Why this Knowledge Base Exists

Most CISSP resources teach Domain 6 as a collection of independent topics:

- Vulnerability Assessment
- Penetration Testing
- Logging
- Auditing
- SIEM
- Software Testing
- Security Metrics

This approach makes the domain difficult to understand because the relationships between these topics are rarely explained.

This knowledge base follows a different philosophy.

Every concept is introduced only after its prerequisites have been established. Instead of memorizing isolated facts, the reader builds a mental model of how security assessment works inside a real organization.

---

# Learning Philosophy

Every concept answers seven questions.

1. What is it?
2. Why does it exist?
3. How does it work?
4. When is it used?
5. What concepts does it depend on?
6. What concepts depend on it?
7. How does ISC² typically test it?

If these seven questions can be answered, the concept has been learned rather than memorized.

---

# Domain 6 in One Sentence

> **Domain 6 provides confidence that security controls are correctly designed, correctly implemented, operating effectively, and producing evidence that management can trust.**

Everything in this domain ultimately supports that single objective.

# How to Study

For every module:

1. Read the concepts in order.
2. Understand the diagrams.
3. Compare similar concepts.
4. Complete the question mappings.
5. Review the module summary.

Avoid memorizing definitions before understanding the relationships between concepts.

---

# Goal

The objective of this knowledge base is not simply to pass the CISSP examination.

Its objective is to develop a structured understanding of Security Assessment and Testing that remains useful long after the exam.
